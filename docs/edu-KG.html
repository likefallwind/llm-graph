Technical Roadmap · EduKG × Graph Memory

面向个性化教育大模型的教育知识图谱技术路线

自动化构建可验证的教育知识图谱，并让图谱在证据、审核和学生交互中持续自进化；同时让图谱与大模型双向交互，支持可解释、可追踪的个性化教学输出。

Automated & Self-Evolving KG · 自动化构图与自进化

从教材、课标、讲义、题库和人工 seed 中自动抽取知识点、题目、资源、误区与关系；所有候选先进入 proposed，经证据校验、双信源复核和治理提交后进入生效图，并在交互证据与审核反馈中持续更新。

KG × LLM Personalization · 图谱与大模型交互

图谱为大模型提供目标知识、先修路径、学生状态、近期证据、活跃误区和可用资源；大模型生成教学反馈后，交互结果继续写回 Evidence Graph，驱动学生状态、资源质量和图谱候选演化。

可信底座 图谱语义 质量闸门 图记忆演化 服务架构

可信教育图谱底座

目标：把高质量教育材料转成稳定、可追溯、可验证、可被教学系统调用的统一 Educational KG，并覆盖知识结构、题目映射、教学资源、误区建模与治理状态。

1.1 自动化构图主流水线

Trusted Sources → Candidate Extraction → Verification → Governed Commit
1可信语料进入
  • 教材 / 课标 / 讲义 / 题库 / 人工 seed 优先
  • 切分章节、题目和解析，冻结 source_id、version、content_hash
  • Wiki / Wikidata / ConceptNet 只作为后续扩展候选源
2候选图谱生成
  • 零 LLM 结构挖掘：目录、课标层级、题库标签产生初始关系候选
  • LLM 有据抽取：只读给定材料，输出 entity / relation / evidence_span
  • 候选包括知识点、技能、题目、资源、误区和关系
3质量验证与裁决
  • 所有 LLM 产物先入 proposed，不直接进入生效图
  • 证据子串校验 + 四级去重 + 粒度对齐，防止误合并和误拆分
  • 结构佐证 × LLM 判断题复核；冲突、弱证据、先修争议进入人工 review
4受控提交与演化
  • 仅 seed + approved 写入 Educational KG 生效子图
  • 每次提交记录 batch_id、review_log、source pointer 和可回滚 patch
  • 审计抽检与交互反馈进入 precision 校准，持续调节自动放行阈值
初始构图阶段先把教材、课标、讲义、题库和人工 seed 中可验证的知识结构、先修关系、题目映射和误区雏形做扎实；资源节点可以进入同一张 Educational KG，但多模态内容先以 source pointer / span / timestamp / region 方式挂接，不作为第一阶段的知识结构来源。Wiki / Wikidata / ConceptNet 不进入初始主通道，主要在 Self-Evolving 阶段用于候选扩展、别名补充和结构佐证。

质量闸门与治理机制

采用证据机械校验、proposed/approved 状态机、双信源复核、规则守卫与人工审核闭环。多模型投票只作为辅助信号，不作为直接入图依据。

3.1 闸门一：证据优先抽取

mechanical evidence gate

Zero-trust IE

  • LLM 只读当前语料块，不允许凭模型记忆补知识
  • 每个实体 / 关系必须绑定 source 与 evidence_in_text
  • evidence 必须通过正文子串校验，不通过则重引一次
  • LLM 输出一律先进入 proposed，不能直接进入生效图
accept_candidate(t) only if:
  evidence_in_text ⊂ source_text
  AND schema_valid(t)
  AND no_hard_constraint_violation(t)
  AND status(t) = proposed

commit(t) only if:
  human_approved OR dual_source_verified

3.2 闸门二：双信源复核

structure × LLM judgment

Verify Before Approval

  • 结构佐证：教材目录、课标层级、题库解析、同章节邻近
  • LLM 判断题：只给证据上下文、候选边、节点定义和图谱邻域
  • 自动批准：结构佐证与 LLM 判断同时支持，且不违反硬约束
  • 人工 review：弱证据、信号冲突、先修争议、粒度争议
LLM 在这里是阅读理解裁判，不是知识源；它只能判断给定材料是否支持候选节点或候选边。

3.3 闸门三：规则守卫

constraints

Hard Constraints

  • Exercise 才能 TESTS Concept / Skill
  • Misconception 不能 PREREQUISITE_OF Concept
  • PREREQUISITE_OF / PART_OF / IS_A 入图前模拟加边查环
  • Resource 节点与 EXPLAINS / REMEDIATES 等边必须有 source pointer / span / timestamp / region
IF edge.type = PREREQUISITE_OF:
  require head.grade_level ≤ tail.grade_level
  require textbook_toc_support OR curriculum_support OR expert_review
  reject if adding edge creates cycle
  cap confidence if edge is inferred

3.4 闭环治理：审核、校准与回滚

review_log × signal → precision

Status Machine

seed / proposed / approved / rejected；所有教学调用和图算法只读取 seed + approved。

Dual-source Verify

结构佐证与文本证据同时成立才自动批准；弱证据、冲突关系和先修争议进入人工 review。

Audit Sampling

对自动批准结果执行抽检；人工推翻样本写入 review_log，成为校准数据。

Calibration

按来源通道、边类型、佐证组合统计 precision，反馈调节自动裁决松紧。

Batch Rollback

每次自动提交记录 batch_id 和 graph patch；发现污染可整批回滚。

3.5 粒度统一

granularity

Chapter

一元二次方程、函数、力学等课程章节或主题单元。

KnowledgePoint

根的判别式、函数单调性、牛顿第二定律等可教学知识点。

Concept / Skill

Δ、实数根、根据 Δ 判断根个数等概念与可操作技能。

Facet / Misconception

相似但不应独立成点的描述降级为 facet；高价值错误模式再升级为 Misconception。

3.6 质量评估指标

offline eval

Entity F1

抽取实体与专家标注实体的 precision / recall / F1，按 Concept、Skill、Misconception 分开统计。

Relation Accuracy

关系类型、方向、证据有效性分别评估，尤其关注 prerequisite 和 tests mapping。

Granularity Consistency

随机抽样检查同层节点粒度是否统一，统计 over-split / over-merge。

Teaching Utility

用图谱辅助诊断、推荐题目、生成提示，看下游教学任务提升。

图记忆与多层次 Self-Evolving

Self-Evolving 不是单一模块，而是多层闭环：Educational KG 在证据和审核反馈中受控扩展，资源节点根据教学使用效果更新映射与质量分，Learner State 随交互高频变化，Governance 根据 review_log 校准自动裁决策略。

4.1 多层次 Self-Evolving 总览

KG structure → resource quality → learner → governance

Layer A · KG Structure Evolution

从 approved 节点发现缺定义、缺先修、缺题目映射、缺误区；优先回到教材 / 课标 / 题库找证据，再用 Wiki / Wikidata / ConceptNet 扩展候选。

Layer B · Resource Node Evolution

在同一张 Educational KG 中，根据资源被调用后的答题改善、提示依赖、错误复发率和教师反馈，更新资源到知识点 / 误区 / 技能的映射与质量分。

Layer C · Learner State Evolution

每次交互写入 Evidence Graph，并更新 mastery、misconception_prob、forgetting_risk、hint_dependency 等学生状态。

Layer D · Governance Evolution

review_log 记录 human / auto 裁决，按来源通道 × 边类型 × 佐证组合统计 precision，自动调节后续放行阈值。

层次触发信号候选产物更新频率是否直接改主图
Educational KG · Structureschema 完整性缺口、教材/题库新术语、审核通过率高的模式CandidateConcept / CandidateRelation / CandidateMisconception低频否，必须 proposed → verified → approved
Educational KG · Resource资源使用效果、资源失效、资源与误区匹配不足CandidateResourceMapping / resource_quality_patch中频否，先更新质量分和候选映射，再受控提交资源边
Learner State答题正确性、ErrorSpan、HintUse、时间衰减StudentConceptState / StudentMisconceptionState高频可以自动更新学生 overlay，不改共享 Educational KG
Governancereview_log、审计抽检、人工推翻率、坏批次回滚threshold_patch / rule_patch / rollback_patch周期性只改裁决策略，不直接新增知识
多层次自进化的关键边界：学生状态可以高频自动变，资源质量可以中频重估，共享 Educational KG 的结构性变更只能低频、证据驱动、版本化合并；治理层负责让自动化程度随 precision 数据变松或变紧。

4.2 外部扩展信息源

Wiki / Wikidata / ConceptNet as expansion sources

Wikipedia

用于补充概念定义、别名、相关页面和跨章节候选。只在图谱发现缺口后定向检索，页面需锁定 revision_id。

Wikidata

用于实体消歧、同义实体合并和结构佐证，例如 subclass / part-of / related entity claim；不能单独决定教学先修。

ConceptNet

用于常识型关联、同义词、上下位和 prerequisite 候选提示；只生成 candidate，不直接写入 Educational KG。

Safety Gate

外部源产物必须回到教材、课标、讲义、题库或人工审核中获得证据支持；找不到教育语料支撑则保留为 proposed 或丢弃。

缺口类型外部源如何参与产物状态入图条件
缺定义 / 缺别名Wikipedia 页面摘要、重定向、同义标题；Wikidata label / aliasCandidateAlias / CandidateDefinition教材或课标术语匹配，且人工或双信源验证通过
缺上位 / 组成关系Wikidata subclass / part-of claim，ConceptNet IsA / PartOfCandidateRelation不违反 schema 和环检测，并有课程结构或教材语料佐证
缺相关概念Wikipedia 内链、分类、ConceptNet RelatedToCandidateConcept回到教育语料中找到 evidence span 后才能进入 verify
缺先修候选ConceptNet prerequisite 类关系、Wikidata 相关 claim 仅作提示CandidatePrerequisite必须由教材目录序、课标层级或专家审核支撑，外部源不能单独批准
外部开放知识源在这里的角色是“扩展候选空间”，不是“替代教育语料”。它们帮助系统知道往哪里找，但所有正式知识仍必须经过 proposed → verify/review → approved 的受控流程。

4.3 教育图记忆闭环

write → update → read → teach → write back

Shared Memory

Educational KG、Pedagogical Rules。知识结构与资源节点低频或中频受控更新，所有学生共享。

Learner Overlay

StudentConceptState、StudentSkillState、StudentMisconceptionState。高频更新。

Evidence Store

Attempt、ErrorSpan、HintUse、Feedback、Assessment。只追加，不覆盖。

1. Interaction Ingestion input

学生答案、草稿步骤、追问、提示使用、作答时长、上下文题目。

raw_event = {student_id, exercise_id, answer, steps, timestamp}

2. Education Memory Writer agentic

抽取涉及知识点、技能、错误步骤、误区候选、证据强度，并写入 Evidence Graph。

Attempt -[:INDICATES_MISCONCEPTION]-> Misconception

3. Learner State Updater controlled

用证据和知识追踪模型更新 mastery、confidence、forgetting_risk、hint_dependency。

state_t+1 = f(state_t, evidence, decay, difficulty)

4. Education Memory Reader graph reader

读取目标知识、先修缺口、最近证据、活跃误区、有效资源，形成教学上下文包。

personalized_subgraph = G[k-hop around target + learner states]

5. Pedagogical Planner policy

选择 hint / explanation / Socratic question / remediation / practice / review。

action = π(teaching_context, learner_state)

6. Tutor LLM + Write-back generation

LLM 负责自然表达；系统再记录反馈内容、学生后续反应和干预效果。

feedback_event → Evidence Graph

Evolution Buffer

从大量证据中发现新误区、新题型、新资源映射、新先修候选。

Validator

规则校验 + 统计支持 + LLM critic + 教师审核 + review_log 留痕。

Graph Patch

通过 patch 合并到 Educational KG，保留版本和回滚路径。

4.4 状态更新机制

not pure LLM
Concept Mastery m' = σ(w₁m + w₂correct - w₃hint - w₄difficulty + w₅evidence_conf)

掌握度由正确性、提示依赖、题目难度和证据置信度共同影响。

Misconception Probability p' = p + λ·evidence_strength - μ·successful_correction

同类错误重复出现则上升；被明确纠正并迁移成功则下降。

Forgetting Risk r = 1 - exp(-Δt / stability)

结合上次接触时间、历史稳定性和复习成功率估计遗忘风险。

信号来自哪里如何影响状态为什么不能只靠 LLM
correctness自动评分 / 规则 / 教师评分直接影响 mastery，但要按题目难度加权。LLM 评分可能不稳定，需要 rubrics 和校验。
error typeWriter 抽取 ErrorSpan映射到 misconception_prob 和 skill_gap。需要证据链，避免把表达问题误判为概念错误。
hint use系统日志影响 hint_dependency 和 mastery 置信度。这是行为信号，不应由 LLM 猜测。
time decay时间戳影响 forgetting_risk 和复习推荐。需要显式时间模型。

存储与服务架构

不要强行“一切进图数据库”。图数据库管理结构关系，向量库管理语义召回，对象存储保留原始资源，事件库负责审计和回放。

5.1 工程组件

serving stack

Graph DB

Educational KG、Evidence Graph、Learner State Graph。支持 k-hop traversal、schema constraint 和跨层查询。

Vector DB

教材片段、视频转写、图片描述、题目解析 embedding，用于候选召回。

Event Store

原始交互、评分结果、模型版本、prompt 版本、状态更新日志，只追加可回放。

Object Storage

PDF、图片、视频、音频、OCR 结果、ASR 文本和切片资源。

Reader Bundle 的作用

  • 读取对象是教学决策子图，包括目标知识、先修缺口和资源映射。
  • 上下文包含学生状态、近期证据、活跃误区和教学策略。
  • 每次 Tutor 输出会继续写回 Evidence Graph，形成长期记忆闭环。
  • 输出格式稳定，便于 Planner 和 Tutor 消费。

Design principle: unified educational KG + student-specific learner overlay + append-only evidence graph + controlled evolution buffer.
