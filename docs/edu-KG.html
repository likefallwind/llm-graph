第一部分：KG 构建

目标：把课程资源转成稳定、可追溯、可验证、可被教学系统调用的教育图谱底座，并补强教育场景所需的 schema 约束、题目映射和误区建模。

1.1 自动化构图主流水线

Extraction → Verification → Integration → Augmentation
0Source Registry
  • 教材 / 课标 / 讲义 / 题库 / 视频 / 图片 / 音频
  • 记录 source_id、grade、subject、version、license
  • 保留 raw asset 与 evidence span
1Multimodal Parsing
  • PDF layout + OCR + table extraction
  • ASR 转写视频/音频
  • CLIP / BLIP / VLM 生成图像语义标签
2Ontology-constrained IE
  • 按 schema 抽取 Concept / Skill / Formula / Exercise
  • LLM 输出 JSON schema，不收自由文本
  • 每个 triple 附 evidence 与 confidence
3Self-Refine
  • Extractor → Critic → Reviser 多轮修正
  • 检查粒度、别名、关系方向、证据缺失
  • 低置信 triple 进入待审池
4Self-Consistency Vote
  • 多模型 / 多 prompt / 多采样投票
  • 保留 vote_score ≥ α 的节点和边
  • 冲突关系进入 conflict queue
5Canonicalization
  • 别名合并：embedding + 字符串 + 课程层级
  • 知识粒度对齐：KP → Concept → Skill
  • 外部锚定：Wikipedia / ConceptNet / 课标术语
6Graph Commit
  • 写入 Domain KG / Resource Graph
  • 生成向量索引与 graph index
  • 记录版本、来源、审核状态、可回滚 diff
推荐不要让 LLM 直接“自由构图”。所有抽取都必须受 ontology、JSON schema、课程层级和证据 span 约束；否则图谱会在粒度和关系语义上迅速漂移。

1.2 构图组件拆解

modules

Extractor

多模态输入 → schema-constrained JSON。输出实体、关系、证据、置信度。

Critic / Verifier

检查三类错误：实体粒度不一致、关系方向错误、无证据 hallucination。

Entity Linker

把 “判别式 / Δ / b²-4ac” 链接到同一 canonical concept。

Graph Integrator

做 duplicate merge、conflict resolution、version diff、graph commit。

2.3 核心关系类型

edge set
子图核心关系技术约束用途
Domain KGPART_OF, PREREQUISITE_OF, REQUIRES_SKILL, TESTED_BY, HAS_MISCONCEPTION关系方向必须通过规则校验；先修边需要课程层级或专家证据。诊断知识缺口、规划学习路径。
Resource GraphEXPLAINS, DEMONSTRATES, REMEDIATES, HAS_HINT, HAS_SOLUTION_STEP多模态资源必须绑定 timestamp/span/region；资源质量动态评分。个性化讲解、补救资源推荐。
Evidence GraphEVIDENCES_CONCEPT, EVIDENCES_SKILL, INDICATES_MISCONCEPTION, USED_HINT每条证据保留 raw_text、model_id、prompt_version、confidence。状态更新、审计、教师复查。
Learner StateHAS_CONCEPT_STATE, HAS_SKILL_STATE, HAS_MISCONCEPTION_STATE, NEXT_TARGET使用状态节点而非直接在边上写 mastery，便于版本化和时间序列。长期学生建模、个性化策略。

质量控制：如何保证图谱不是“LLM 幻觉图”

采用多模型抽取、自反馈修正、自一致性投票、外部知识增强与人工审核闭环，并额外加入教育粒度控制和关系方向校验。

3.1 多模型抽取

ensemble

LLM Ensemble

  • 模型 A：高精度抽取
  • 模型 B：召回补充
  • 模型 C：中文教育术语校验
  • temperature = 0 用于稳定抽取
vote_score(t) = count(agree(t)) / N
accept(t) if vote_score(t) ≥ α
review(t) if β ≤ vote_score(t) < α
reject(t) if vote_score(t) < β

3.2 粒度统一

granularity

Granularity Ladder

  • Chapter：一元二次方程
  • KnowledgePoint：根的判别式
  • Concept：Δ / 实数根
  • Skill：根据 Δ 判断根的个数
  • Misconception：Δ<0 表示方程无意义
实体合并不能只靠 embedding，相似但粒度不同的节点不能合并。

3.3 关系校验

constraints

Hard Constraints

  • Exercise 才能 TESTS Concept / Skill
  • Misconception 不能 PREREQUISITE_OF Concept
  • PREREQUISITE_OF 不能形成明显循环
  • Resource 必须有 source pointer
IF edge.type = PREREQUISITE_OF:
  require head.grade_level ≤ tail.grade_level
  require evidence OR expert_review
  reject cycles longer than 1

3.4 质量评估指标

offline eval

Entity F1

抽取实体与专家标注实体的 precision / recall / F1，按 Concept、Skill、Misconception 分开统计。

Relation Accuracy

关系类型、方向、证据有效性分别评估，尤其关注 prerequisite 和 tests mapping。

Granularity Consistency

随机抽样检查同层节点粒度是否统一，统计 over-split / over-merge。

Teaching Utility

用图谱辅助诊断、推荐题目、生成提示，看下游教学任务提升。

第二部分：KG Self-Evolving

核心是图记忆闭环：每次学习交互不是简单日志，而是被 Writer 结构化为图记忆，由 Updater 更新学生状态，由 Reader 读取个性化子图，再由 Tutor 生成教学动作。

4.1 教育图记忆闭环

write → update → read → teach → write back

Shared Memory

Domain KG、Resource Graph、Pedagogical Rules。低频更新，所有学生共享。

Learner Overlay

StudentConceptState、StudentSkillState、StudentMisconceptionState。高频更新。

Evidence Store

Attempt、ErrorSpan、HintUse、Feedback、Assessment。只追加，不覆盖。

1. Interaction Ingestion input

学生答案、草稿步骤、追问、提示使用、作答时长、上下文题目。

raw_event = {student_id, exercise_id, answer, steps, timestamp}

2. Education Memory Writer agentic

抽取涉及知识点、技能、错误步骤、误区候选、证据强度，并写入 Evidence Graph。

Attempt -[:INDICATES_MISCONCEPTION]-> Misconception

3. Learner State Updater controlled

用证据和知识追踪模型更新 mastery、confidence、forgetting_risk、hint_dependency。

state_t+1 = f(state_t, evidence, decay, difficulty)

4. Education Memory Reader graph reader

读取目标知识、先修缺口、最近证据、活跃误区、有效资源，形成教学上下文包。

personalized_subgraph = G[k-hop around target + learner states]

5. Pedagogical Planner policy

选择 hint / explanation / Socratic question / remediation / practice / review。

action = π(teaching_context, learner_state)

6. Tutor LLM + Write-back generation

LLM 负责自然表达；系统再记录反馈内容、学生后续反应和干预效果。

feedback_event → Evidence Graph

Evolution Buffer

从大量证据中发现新误区、新题型、新资源映射、新先修候选。

Validator

规则校验 + 统计支持 + LLM critic + 教师审核 + A/B 效果验证。

Graph Patch

通过 patch 合并到 Domain KG / Resource Graph，保留版本和回滚路径。

4.2 状态更新机制

not pure LLM
Concept Mastery m' = σ(w₁m + w₂correct - w₃hint - w₄difficulty + w₅evidence_conf)

掌握度由正确性、提示依赖、题目难度和证据置信度共同影响。

Misconception Probability p' = p + λ·evidence_strength - μ·successful_correction

同类错误重复出现则上升；被明确纠正并迁移成功则下降。

Forgetting Risk r = 1 - exp(-Δt / stability)

结合上次接触时间、历史稳定性和复习成功率估计遗忘风险。

信号来自哪里如何影响状态为什么不能只靠 LLM
correctness自动评分 / 规则 / 教师评分直接影响 mastery，但要按题目难度加权。LLM 评分可能不稳定，需要 rubrics 和校验。
error typeWriter 抽取 ErrorSpan映射到 misconception_prob 和 skill_gap。需要证据链，避免把表达问题误判为概念错误。
hint use系统日志影响 hint_dependency 和 mastery 置信度。这是行为信号，不应由 LLM 猜测。
time decay时间戳影响 forgetting_risk 和复习推荐。需要显式时间模型。

存储与服务架构

不要强行“一切进图数据库”。图数据库管理结构关系，向量库管理语义召回，对象存储保留原始资源，事件库负责审计和回放。

5.1 工程组件

serving stack

Graph DB

Domain KG、Resource Graph、Evidence Graph、Learner State Graph。支持 k-hop traversal 和 schema constraint。

Vector DB

教材片段、视频转写、图片描述、题目解析 embedding，用于候选召回。

Event Store

原始交互、评分结果、模型版本、prompt 版本、状态更新日志，只追加可回放。

Object Storage

PDF、图片、视频、音频、OCR 结果、ASR 文本和切片资源。

teaching context bundle
{
  "target": {
    "exercise_id": "math_q_1024",
    "concepts": ["判别式", "实数根"],
    "skills": ["根据Δ判断根的个数"]
  },
  "learner_state": {
    "判别式": {"mastery": 0.54, "confidence": 0.78},
    "根据Δ判断根的个数": {"mastery": 0.47}
  },
  "active_misconceptions": [
    {"name": "认为所有二次方程都有两个实数根", "prob": 0.71}
  ],
  "recent_evidence": ["连续两次没有主动计算Δ", "提示后可以代入公式"],
  "recommended_action": "socratic_hint",
  "allowed_resources": ["hint_12", "visual_explanation_03"]
}

Reader Bundle 的作用

  • 读取对象是教学决策子图，包括目标知识、先修缺口和资源映射。
  • 上下文包含学生状态、近期证据、活跃误区和教学策略。
  • 每次 Tutor 输出会继续写回 Evidence Graph，形成长期记忆闭环。
  • 输出格式稳定，便于 Planner 和 Tutor 消费。

Design principle: shared domain-resource graph + student-specific learner overlay + append-only evidence graph + controlled evolution buffer.
