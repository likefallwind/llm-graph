0. 核心原则（四条不变式）

所有模块改动都不得破坏这四条。它们是本方案与"LLM 自由构图"路线的根本分界。

① LLM 是抽取器，不是知识源

图谱内容必须落地到本地语料快照。LLM 只做判断题（边类型、粒度、证据摘取），不做默写题（凭记忆生成知识）。 LLM 记忆只允许出现在"往哪看"（expand 提假设名，须经语料验证，找不到来源页即丢弃），不进图谱内容。

② 证据机械可校验

每个概念、每条边必须附来源正文原文摘录（≤60 字），evidence_in_text 子串校验不过整条丢弃。 LLM 完全可能编造一句"原文"，这一行校验是后续一切佐证的地基。英文教材以翻译后中文快照为准，入库即冻结。

③ 置信度 = 独立信源是否一致

不用 LLM 自报数值置信度分流（自报置信度校准差，错的时候照样报高分）。 结构佐证（互链 / RefD / 教材目录序 / Wikidata）与 LLM 判断题复核是两个独立信源，双重一致才允许自动裁决。

④ 全裁决留痕，闭环校准

人工与自动裁决全部写 review_log；calibrate 按（通道 × 类型 × 佐证组合）统计 precision， 自动放行用 review --audit 抽检推翻率。高精度组合放宽、低的收紧——松紧由数据决定，不由直觉决定。

2. 数据模型与边语义

SQLite 单库；status 字段兼作审核队列，是半自动进化的骨干。层次不用节点属性表示，用类型化的边表示。

2.1 表结构

kg/db.py

图谱主体

nodes: id, name(唯一), aliases[], definition, facets[], status, source, embedding, created_at edges: id, src, dst, type, confidence, rationale, source, status, UNIQUE(src,dst,type)

语料层

corpus: lang, page_id, title, revision_id, text, redirects[], categories[], links[] node_page: node_id → (lang, page_id) doc_sections: book, sec_id, ord, title, text, content_hash

复核与校准

review_log: 每次裁决留痕 (动作, 通道, human/auto, batch_id ← 新增) review_signals: 结构佐证 + LLM 复核结论 page_qid / wikidata_claims: 缓存

防重复

ingest_log: (锚点, 来源版本) 唯一 同版本不重复提取 add_edge 入库拦截: related_to 反向重复 插入前即拒 
状态机（status 兼作审核队列）：

seed ──────────────┐
                   ├──> 生效（所有图算法、守卫、教学导出只看这两种）
proposed ─审核a──> approved
    │
    └──审核r/m/d──> rejected（m 合并：边转移到目标节点后 reject；d 降级：文本进父节点 facets）

生效的两条路：人工 review；或 verify --apply 双重一致自动裁决。两条路都必写 review_log。

2.2 边类型语义

db.EDGE_TYPES · 4 种
类型语义约束
is_aA 是 B 的一种生效子图无环（守卫）
part_ofA 是 B 的组成部分生效子图无环（守卫）
prerequisite_of不懂 A 就无法理解 B（教学先修）必须 DAG；唯一允许推断的边——rationale 加 [推断] 前缀、confidence 封顶 0.6
related_to受限关联（封闭枚举）只允许三种 kind：同题替代 / 演化启发 / 教学对比；无合法 kind 代码层直接丢弃；语义对称，入库拦反向重复
先修是最有教学价值、却最不可抽取的边（维基正文几乎从不写"先学 A 再学 B"）。对策：允许受控推断 + 教材目录序（toc）作为独立信源交叉验证——教材目录天然编码教学先修顺序，比 LLM 零样本判先修硬得多。

2.3 节点粒度：facet 升级规则

粒度随图长出来，不预先设计

规则

  • 候选知识点默认不是节点，而是父节点的 facet（≤12 字名词短语，如 softmax 的「温度参数」）
  • 只有当别的知识点需要单独依赖它（要给它连边）时，才升级为独立节点
  • 算法内部步骤 / 参数 / 公式细节一律进 facets；拿不准宁可放 facets
  • 守卫 facet_shadows 做逆向检查：facet 与已有节点重名 → 已被升级，父节点应清理
与 edu-KG 方案的 Granularity Ladder（Chapter→KP→Concept→Skill 固定分层）相比，本方案的粒度是演生的。吸收其要点落在去重环节：相似但粒度不同的节点不能合并为别名——dedup 第 4 级 LLM 裁决显式区分「同一概念」与「是对方的细粒度成分」，后者走降级 facet（§4 A5）。

3. 三道质量闸门

回答"如何保证图谱不是 LLM 幻觉图"：提取时机械校验，复核时双独立信源，事后守卫 + 审计 + 校准。每一道都可被数据检验。

3.1 闸门一 · 提取时

机械规则，零信任

入库前逐条过滤

  • evidence 必须是来源正文子串（规范化后逐段校验），不过整条丢弃
  • related_to 无合法 kind 直接丢弃（dropped_related）
  • facet 长度 >16 字符不收
  • 推断先修 confidence 封顶 0.6，与有据边区分信任层级
  • 正文 <300 字不算有效来源
  • 宁缺毋滥：--limit 每页产出预算总闸门

3.2 闸门二 · 复核时

双独立信源一致才动
对象自动批准自动拒绝
节点 LLM 判「独立概念」＋ 名字精确命中语料页 ＋ 语料页与图谱邻居有重叠（UMAP 撞名补丁） 无（拒绝 / 降级 facet 一律人工）
边 LLM「支持」方向无疑 ＋ ≥1 结构佐证（互链 / Wikidata / toc=+1）；先修边另要求 refd 与 toc 都不反对 ＋ 无环类型模拟加边不成环（守卫前移为放行拦截） LLM「不支持」＋ 零结构佐证

low 级语料的条目一律不自动裁决；其余不满足条件的全部留人工。整次运行共用一个批次号（可 kg rollback 整批撤销）。LLM 复核只做阅读理解题（只给证据上下文、双方定义、图谱邻域），prompt 明令禁用模型记忆——复核时 LLM 也不是知识源。

3.3 闸门三 · 事后

守卫 + 审计 + 校准

持续运行

  • guards：先修 / is_a / part_of 三色 DFS 环检测、先修传递冗余、正反向同型边、孤儿、facet 重名——只报告不自动修，裁决权在人
  • review --audit N：抽检自动放行，推翻率进 calibrate
  • calibrate：人工裁决 precision 按（通道 × 类型 × 佐证组合）分桶，n<3 标注样本不足
  • kg rollback：坏批次按 batch_id 整批退回 proposed 重审
UMAP 事故（2026-07-05）与已落地的对策（2026-07-06）：多义短名 / 缩写词可被重定向劫持到无关页面（zh「UMAP」→「亞太大學交流會」），此时「名字精确命中」天然成立，LLM 复核只读到错误页面材料也会判「独立概念」——两个信源被同一错误来源污染，不再独立。对策已实现：A2 邻居重叠守卫（错误页面不会提到真概念的图谱邻居，无重叠不放行且在 review 队列排进信号冲突层）＋ A1 批次回滚（放行错了可整批撤销）。该事故的人工 reject 已留痕 review_log，恰好成为一条自动放行被推翻的校准数据。

5. 明确不采纳的部分

记录理由与重新考虑的触发条件，避免同样的讨论重复发生。

✕ 多模型 ensemble 投票

理由：多个 LLM 的错误相关（同一训练分布），vote_score 不是独立信源；成本 ×N。结构佐证（人类校对过的维基结构 / 教材目录）才是真独立信号，且有 calibrate 闭环。
重新考虑：出现能力互补、成本低一个数量级的第二模型时，作召回补充而非投票。

✕ LLM 自报置信度分流

理由：自报数值置信度校准差，错的时候照样报高分——plan.md 已否决的设计决策。置信度 = 独立信源是否一致。
重新考虑：不考虑。

△ 图库/向量库/事件库/对象存储四件套

理由：当前量级（数百节点、单人审核）SQLite 一体化完全够用，拆存储是过早优化。review_log 已承担事件留痕职能。
重新考虑：节点 >10 万、或多学生在线服务上线时。

△ 多模态解析（OCR / ASR / VLM）

理由：文本语料（维基 + 四本教材）的质量瓶颈尚未触及，多模态投入产出比低。
重新考虑："包括所有知识"的远期目标推进到视频课程资源、或教学接口需要图示资源时。

6.2 日常进化循环

kg CLI
kg corpus crawl              # 补齐生效节点的页面（新节点批准后跑一次）
kg corpus grow --limit 10    # 沿内链扩语料（可选）
kg docs fetch <book>         # 教材语料增量抓取
kg docs translate <book>     # 英文源预翻译（可选，不跑则 lazy 翻译）
kg mine aliases / categories / wikidata   # 零 LLM 结构挖掘，随时可跑
kg ingest --batch 3 --doc    # 教材缺口驱动批量提取（high 级语料，优先）
kg ingest --batch 3          # 维基缺口驱动批量提取（mid 级，覆盖面兜底）
kg expand                    # 假设生成器补盲区（辅助）
kg verify --limit 20 --apply # 双信源复核 + 双重一致自动裁决（输出批次号）
kg review                    # 人工裁决剩余队列；--audit N 抽检自动放行
kg rollback [batch_id]       # 坏批次整批退回 proposed（不带参数列批次）
kg calibrate                 # 看各通道×佐证组合 precision，调自动裁决松紧
kg check → kg viz            # 守卫 + 可视化

Design principle: 分级语料快照为地基（教材/教案 > 维基 > 弱校对）· 双独立信源为置信度 · 全裁决留痕为校准 · 批次可回滚 · 人工从守门员到抽检员
